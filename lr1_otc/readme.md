1. Класс-итератор с __iter__ и __next__
Принцип работы: объект хранит текущие значения (a, b) и счётчик. Метод __next__ возвращает следующее число, обновляя состояние. При достижении max выбрасывается StopIteration. Метод __iter__ возвращает сам объект, делая его итератором.

Особенности:

Эффективен для последовательного обхода (каждое число вычисляется один раз).

Позволяет легко ограничить количество элементов через max.

Не поддерживает произвольный доступ (нельзя получить f[5], только последовательно).

Ограничения: однопроходный итератор; после исчерпания (или вручную) его нельзя «перезапустить» без создания нового экземпляра.
2. Класс с __getitem__ (контейнерный протокол)
Принцип работы: при обращении по индексу вычисляет число Фибоначчи с нуля, циклически пересчитывая последовательность до нужного индекса. Если задан параметр max, то доступ к индексам >= max вызывает IndexError, что позволяет использовать объект в цикле for (Python автоматически перебирает индексы 0,1,2… до первого IndexError).

Особенности:

Поддерживает произвольный доступ по индексу (как у списка).

Каждое обращение пересчитывает последовательность заново → неэффективно при частых запросах.

Ограничение max управляет как доступом по индексу, так и длиной итерации.

Ограничения: не сохраняет состояние между вызовами; для длинных последовательностей или многих обращений неоптимален по времени.
3. Корутина (генераторная функция с yield)
Принцип работы: функция, использующая yield для возврата очередного числа, сохраняет своё состояние между вызовами. Для управления используется метод .send(), позволяющий передавать произвольные значения внутрь генератора (в примере они выводятся, но не влияют на последовательность).

Активация: обязательный первый вызов next() или .send(None), иначе .send() с не‑None значением вызовет ошибку.

Особенности:

Бесконечная последовательность (нет встроенного ограничения).

Полноценное двустороннее взаимодействие (можно передавать данные, управлять поведением).

Удобна для потоковой обработки и интерактивных сценариев.

Ограничения: требует явной активации; не поддерживает произвольный доступ по индексу.
